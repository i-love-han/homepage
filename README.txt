===============================================
          윤형삼 홈페이지 사용 설명서
===============================================

이 문서는 홈페이지를 관리하는 방법을 설명합니다.


===============================================
1. 이미지 관리 (자동 반영!)
===============================================

images 폴더에 이미지를 업로드하면 자동으로 반영됩니다!
별도의 설정 파일 수정이 필요 없습니다.

[폴더 구조]
images/
├── gallery/    → 갤러리 이미지들
├── main/       → 메인 배경 이미지 (첫 번째 파일 사용)
├── people/     → 소개 섹션 프로필 이미지 (첫 번째 파일 사용)
├── popup/      → 팝업 이미지들 (이미지 있으면 자동 팝업)
└── no-image.svg → 이미지 없을 때 표시되는 기본 이미지

[갤러리 이미지]
- 위치: images/gallery/
- 파일명에 숫자가 있으면 숫자 기준 내림차순 정렬
  예: 12.jpg → 11.jpg → 10.jpg → ... → 1.jpg
- 6개씩 한 페이지로 슬라이드됨
- 자동 슬라이드 시간은 content.txt에서 조절 가능

[메인 배경 이미지]
- 위치: images/main/
- 첫 번째 이미지가 메인 화면 배경으로 사용됨
- 권장: 고해상도 이미지 (1920x1080 이상)

[프로필 이미지]
- 위치: images/people/
- 첫 번째 이미지가 소개 섹션에 표시됨
- 권장 비율: 4:5 세로형

[팝업 이미지]
- 위치: images/popup/
- 이미지가 있으면 사이트 접속시 팝업으로 자동 표시
- 이미지가 없으면 팝업이 나타나지 않음
- 여러 이미지 가능 (파일명 순서대로 표시)
- 사용자가 "일주일동안 열지 않기" 체크하면 7일간 안 나옴


===============================================
2. content.txt 수정하기
===============================================

content.txt 파일을 수정하면 홈페이지의 텍스트 내용이 변경됩니다.

[형식]
항목이름=내용

[줄바꿈]
\n 을 사용하면 줄바꿈이 됩니다.
예: 메인 내용=첫번째 줄\n두번째 줄

[섹션 구분선]
========== 섹션명 ========== 형태의 줄은 무시됩니다.
보기 좋게 정리하는 용도로 사용하세요.

[수정 가능한 항목들]

● 기본 설정
  - 브라우저 타이틀 : 브라우저 탭에 표시되는 제목
  - 헤더 제목 : 좌측 상단 로고 텍스트

● 라벨 (상단 메뉴 + 섹션 제목)
  - 홈 라벨 : 홈 메뉴 텍스트
  - 소개 라벨 : 소개 메뉴 및 섹션 제목
  - 갤러리 라벨 : 갤러리 메뉴 및 섹션 제목
  - 연락처 라벨 : 연락처 메뉴 및 섹션 제목

● 메인 화면
  - 메인 제목 : 메인 화면 큰 제목
  - 메인 내용 : 메인 화면 부제목 (\n으로 줄바꿈)

● 소개 섹션
  - 소개 제목 : 소개 영역 제목
  - 소개 내용 : 소개 영역 본문 (\n으로 줄바꿈)

● 갤러리 섹션
  - 갤러리 제목 : 갤러리 부제목
  - 갤러리 스크롤시간 : 자동 슬라이드 시간 (초 단위, 예: 3)

● 연락처 섹션
  - 연락처 제목 : 연락처 부제목
  - 연락처 이메일 : 이메일 주소 (클릭시 메일 앱 열림)
  - 연락처 전화 : 전화번호 (클릭시 전화 앱 열림)
  - 연락처 위치 : 주소 (클릭시 카카오맵 검색)

● 푸터
  - 푸터 내용 : 하단 저작권 문구


===============================================
3. 변경사항 반영하기
===============================================

1. 파일 수정/이미지 업로드 후 저장
2. Git 커밋 & 푸시
3. GitHub Pages에 자동 반영됨 (1-2분 소요)


===============================================
4. 주의사항
===============================================

- content.txt는 UTF-8 인코딩으로 저장하세요
- 이미지 파일명에 한글이나 특수문자는 피하세요
- 이미지 용량이 크면 로딩이 느려집니다 (1MB 이하 권장)


===============================================
5. 문제 해결
===============================================

[이미지가 안 보여요]
→ GitHub에 push 했는지 확인
→ 파일 확장자(.jpg, .png 등)가 정확한지 확인
→ 파일명에 특수문자가 있는지 확인

[팝업이 안 나와요]
→ images/popup/ 폴더에 이미지가 있는지 확인
→ "일주일동안 열지 않기"를 체크했다면 브라우저 캐시 삭제
→ 또는 시크릿 모드로 접속

[내용 수정이 반영 안 돼요]
→ git push 했는지 확인
→ 브라우저 새로고침 (Ctrl+Shift+R 강력 새로고침)
→ 1-2분 기다린 후 다시 시도

[GitHub API 오류 (403 에러)]
→ GitHub API 사용량 제한일 수 있음 (시간당 60회)
→ 1시간 후 다시 시도
